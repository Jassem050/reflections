When would you want to use a remote repository rather than keeping all your work local?
Remote repository is neccessary to store all the files remotely where other members of a team can 
access or edit files or create seperate branches.

Why might you want to always pull changes manually rather than having Git automatically stay up-to-date 
with your remote repository?
Unless a change is made in remote repo that are not in local, there is no need to pull.

Describe the differences between forks, clones, and branches. When would you use one instead of another?
Fork is to create a copy of repo in github.
Clone is to create a copy of repo in local machine.
Branches are labled commits of repo that are different from master.
Fork is used to copy repo from another github account to out github instead of clone.

What is the benefit of having a copy of the last known state of the remote stored locally?
To know the status of the remote repo and local and to remove conflicts.
